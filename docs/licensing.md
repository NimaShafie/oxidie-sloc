# Licensing Notes

This repository currently uses **AGPL-3.0-or-later**.

## Why keep AGPL for now

AGPL keeps the repository genuinely open source while still allowing the maintainer to make money from the project through:
- paid support
- consulting
- hosted offerings
- private extensions or add-ons in separate codebases
- negotiated commercial agreements

## What AGPL does not do

AGPL does not stop everyone else from using the public code commercially. It requires them to comply with AGPL terms, including making corresponding source available for modified network-facing deployments.

## If tighter control is needed later

If the goal changes from open source to source-available with stronger commercial restrictions, the project can move to a different licensing model for future versions before significant third-party contributions accumulate.

That kind of switch should be done carefully and ideally before a large outside contributor base develops.

## Practical recommendation

For now:
- keep the repository name simple as `oxide-sloc`
- keep the public code under AGPL
- avoid the phrase "Community Edition" in the product name
- use separate commercial terms only when there is a real paid offering to describe
